PLANS AND CONSTRAINTS

1. INITIAL PLAN

My initial plan was to use highly available API with ease to use data format (json) like https://openweathermap.org/api or https://www.weatherapi.com/api-explorer.aspx. But, as expected, since i am Indonesian i feel like the urge to use the data provided by BMKG (Badan Meteorologi dan Geofisika) instead of aforementioned API. Life is not that easy, i found several constraints which force me to use another method and tech stack to tackle the problems.

2.CONSTRAINTS

  1. BMKG data is available in xml format containing many informations, i need to extract them first. Here is the example of bmkg data in xml format
  2. BMKG data is available only for 3 days range (today,tomorrow,and the day after tomorrow) . This data also replaced every day. Therefore, it is impossible for me to get a lot of data without saving the file every single day.
  3. BMKG website sometimes down for several hours. This happen more than once during May 2023.
  4. Due to the nature of the task, i need to use workflow orchestration tool, such as airflow, therefore i also need to ensure that my airflow to standby 24 hours. I've tried using aiflow with AWS EC2 free tier VM, but the VM is often lagging, therefore i use GCP with 300 dollars credit in order to use a better VM . The credits will expired in 31 July 2023, so does this website haha..
  5. I also plan to use map as a plot, my preferred map style is available in mapbox. But this website has rate limiting API, so the service might not available if the rate limits are exceeded.

The methods which incorporate these constraints as requirements are available in "methods" section
